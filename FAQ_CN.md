## 系统功能问题

### Q: 系统有哪些主要功能？

**A: 三大核心功能：**

1. **员工内部价格系统** 🏷️
   - 酒店员工享受商品折扣价
   - 支持固定员工价和百分比折扣两种方式
   - 自动根据用户角色计算价格

2. **电子优惠券系统** 🎫
   - 满减优惠券（如满50减10）
   - 折扣优惠券（如8折）
   - 固定金额优惠券
   - 支持优惠券领取、使用、过期管理

3. **双端支付支持** 💳
   - 支持微信支付
   - 支持支付宝支付
   - 结账时可选择支付方式

### Q: 如何使用优惠券？

**A: 优惠券使用流程：**
1. 登录系统
2. 进入"优惠券"页面
3. 点击"领取"按钮领取优惠券
4. 添加商品到购物车
5. 结账时选择可用的优惠券
6. 系统自动计算优惠后的价格

### Q: 员工折扣和优惠券可以同时使用吗？

**A: 可以！** 
- 员工首先享受员工折扣价
- 然后可以在员工折扣价基础上再使用优惠券
- 订单会显示完整的价格明细：
  - 原始总额
  - 员工折扣金额
  - 优惠券折扣金额
  - 最终支付金额

**示例**：
```
商品原价：¥7.00（2瓶可乐 @ ¥3.50）
员工折扣价：¥5.60（2瓶 @ ¥2.80，享受20%折扣）
优惠券折扣：-¥1.12（使用20%折扣券）
最终支付：¥4.48
```

## 技术问题

### Q: 如何运行这个项目？

**A: 运行步骤：**

**1. 数据库设置**
```bash
mysql -u root -p
CREATE DATABASE hotel_vending;
USE hotel_vending;
SOURCE backend/src/main/resources/schema.sql;
```

**2. 启动后端**
```bash
cd backend
mvn spring-boot:run
```
后端运行在: http://localhost:8080
API文档: http://localhost:8080/api/doc.html

**3. 启动前端**
```bash
cd frontend
npm install
npm run dev
```
前端运行在: http://localhost:3000

### Q: 需要安装哪些软件？

**A: 环境要求：**
- Java 17 或更高版本
- Node.js 16 或更高版本
- MySQL 8.0 或更高版本
- Redis 6.0 或更高版本
- Maven 3.6 或更高版本

### Q: API 文档在哪里？

**A: API 文档地址：**
- 启动后端服务后访问：`http://localhost:8080/api/doc.html`
- 使用 Knife4j (Swagger) 提供的交互式 API 文档
- 可以直接在文档中测试 API 接口

## 故障排除

### Q: 无法登录怎么办？

**A: 检查以下几点：**
1. 确认数据库已正确导入（运行 schema.sql）
2. 确认后端服务已启动（端口 8080）
3. 确认前端服务已启动（端口 3000）
4. 使用正确的用户名和密码（admin / 123456）
5. 检查浏览器控制台是否有错误信息

### Q: 提示"网络错误"怎么办？

**A: 可能的原因：**
1. 后端服务未启动或端口被占用
2. 前端配置的后端地址不正确
3. CORS 跨域配置问题
4. 防火墙阻止了连接

**解决方法：**
- 确认后端服务运行在 8080 端口
- 检查前端 vite.config.ts 中的代理配置
- 查看后端日志是否有错误

### Q: 数据库连接失败怎么办？

**A: 检查以下配置：**
1. MySQL 服务是否已启动
2. 数据库名称是否正确（hotel_vending）
3. 用户名密码是否正确
4. 数据库端口是否为 3306
5. 时区设置是否正确
